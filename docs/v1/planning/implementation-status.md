## Beta Transport Status

This note captures the published beta state for human review. The detailed matrix below remains the implementation-area tracker.

Done:

- `grape-context@1.0.0-beta.6` is published on npm under the `latest` and `beta` dist-tags. `1.0.0-beta.0` remains the first beta tag and first committed published-package benchmark baseline.
- Alpha releases `0.1.0-alpha.1` through `0.1.0-alpha.3` remain on npm under the `alpha` dist-tag as historical prereleases.
- The context transport wedge is proven through CLI and MCP `grape_get_context`.
- MCP stdio smoke exists.
- Same-session two-turn omission works when task/session identity is stable.
- `PINNED`, `OMIT_UNCHANGED`, `RESTORE_AVAILABLE`, and `INVALIDATE_PREVIOUS` are implemented in the current transport slice.
- Branch/dependency invalidation, explicit reset invalidation, and omitted restore are proven by behavior coverage.
- Packaged install smoke now asserts exact tarball selection, installed package metadata, CLI help/init/two-turn compile, CLI omitted-item restore, task/session mismatch recovery, reset recovery, MCP `initialize` / `tools/list`, two-turn context diffing, and omitted-item restore.
- Alpha e2e smoke now uses a repo-local npm cache, asserts exact tarball/package metadata, checks restore hints, and exercises installed MCP stdio setup.
- Branch-switch and stale-source fixture metadata is aligned to invalidation benchmark behavior.
- Session reset fixture benchmark proves explicit reset emits `INVALIDATE_PREVIOUS`, sends new current context, and avoids `OMIT_UNCHANGED` on the reset turn.
- Restore-path golden tests cover `RESTORE_AVAILABLE` restore IDs, session-bound lookup, restored body shape, and MCP no-root-path output.
- Dedicated task/session mismatch exit classification returns code `6` from CLI compile/diff-context instead of the storage/schema bucket.
- Package-lock metadata is aligned to `1.0.0-beta.6`. Global install smoke is expected to use registry-installed `@beta`.
- Storage repository ownership is split into table-family directories without schema or behavior changes.
- The external benchmark workspace reports scripted transport scenarios passing under its documented methodology and stable task/session contract. Those are local fixture results, not production performance evidence.
- GitHub Actions run `27449834015` passed `npm run check` on Ubuntu, macOS, and the pinned `windows-2025-vs2026` runner, plus the beta smoke job with `benchmark:run`, `e2e:alpha`, and `beta:client-trial`.

Now:

- Record human MCP client trials from `beta-trial-checklist.md` when release policy requires IDE UI proof beyond automated `beta:client-trial`.
- Repeat formal benchmark comparison runs after the first committed post-beta published-package baseline, and treat current results as fixture-level evidence only.
- Keep future version bumps, tags, GitHub releases, npm publishes, and dist-tag changes approval-gated.

Next:

- Beta candidate only after install/connect/recover flows are boring across clean consumer repos.

Soon:

- Better TypeScript compiler retrieval and exact-span ranking.
- Stronger observed-run trust workflows beyond the narrow result claim.
- Stronger trust/proof depth, broader benchmark fixtures, and better dirty-worktree/session handling.
