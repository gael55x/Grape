# V1 Implementation Status

This matrix tracks implementation against the final V1 core-pipeline acceptance areas. It is an implementation aid, not a substitute for working code, tests, docs, review, and pushed commits.

Status values:

- Not started
- In progress
- Implemented
- Tested
- Documented
- Reviewed
- Done

| Area | Status | Current evidence | Remaining work |
|---|---|---|---|
| CLI setup flow | Tested | `grape init --connect`, `grape help`, `grape compile --task <text>`, `grape status`, `grape doctor`, `grape mcp --print-config`, and `grape mcp --stdio` have thin adapters and behavior tests. | Broaden CLI inspection commands after artifact/query flows exist. |
| Local bootstrap | Tested | `grape init --connect` creates `.grape/`, writes config, applies SQLite migrations, persists a Git snapshot, persists source/source-rejection evidence rows, persists lightweight file index rows, and excludes `.grape/` through `.git/info/exclude`. `grape compile --task <text>` auto-bootstraps the same local state if needed. | Add candidate rules, framework/package-manager detection, and recovery tests for partial bootstrap. |
| Project scanner | Tested | Git-backed snapshot captures branch, commit, dirty paths, Git-visible files, file hashes, source-kind classification, Git-visible ignored/private path rejections, `.gitignore`, and local privacy ignore files before reading allowed file bytes. Ignored untracked paths are skipped rather than enumerated. | Add size/binary handling, approval records, staged/untracked source-scope splitting, and explicit scan diagnostics. |
| File indexing | Tested | Allowed snapshot files are indexed into source-anchored module/symbol nodes and `contains`/`imports` edges with confidence and discovery method; private/ignored sources are excluded; symlinks, binary files, and snapshot-hash mismatches are skipped. | Add FTS/source indexes, broader language extraction, symbol body ranges, and explicit blind-spot reporting. |
| Dependency tracking | Tested | Durable context build persists artifact dependency manifests and invalidates stale sent items when manifest hashes change. Repository-derived artifacts include repo snapshot, worktree, source/config/lockfile, and symbol relationship dependencies. CLI compile now writes dependency-tracked JSON/Markdown scaffold artifacts. | Connect the repository-derived compile service to MCP retrieval and final V1 artifact schema. |
| Branch awareness | Tested | Repo snapshots persist branch, head commit, dirty state, worktree hash, and sent ledgers include branch/commit identity. | Add branch-switch invalidation across active context sessions and current-valid claim filtering. |
| Session diffing | Tested | Durable build proof persists session-scoped sent/omitted ledgers, safe unchanged omission, restore metadata, pinned resend, and stale manifest invalidation. | Add restore lookup command/tool and session reset/branch-switch behavior in product flow. |
| Task-specific context pack generation | In progress | `grape compile --task <text>` and MCP `grape_get_context` compile from real repository input, persist durable session diff rows, write JSON/Markdown scaffold artifacts, and return NEW/PINNED/OMIT_UNCHANGED/RESTORE_AVAILABLE context pack items across repeated session calls. Risk overlays fail closed with an unsafe reason until exact spans exist. | Add exact-span selection, final V1 JSON schema, final Markdown contract, restore lookup, and stronger compiler policies. |
| Proof/evidence model | Tested | Shared trust/proof/evidence shapes exist; snapshot source ingestion persists allowed source records and privacy-safe source rejections without reading rejected bytes. | Implement proof validation, belief gate, candidate rejection, durable promotion rules, and exact proof-span support. |
| Context artifact format | In progress | In-memory artifact guards require dependency manifests, section hashes, dependency refs, exact source refs, proof refs for exact claims, and blocked-redaction rejection. Repository-derived artifacts now include repo-state, source-manifest, file-relationship, and blind-spot sections from real repo inputs, and CLI compile writes scaffold JSON/Markdown under `.grape/artifacts/`. | Promote to final V1 artifact schema and final Markdown contract. |
| MCP integration surface or contract | Tested | `grape mcp --stdio` implements framed JSON-RPC stdio for `initialize`, `tools/list`, `tools/call`, `grape_get_context`, and `grape_get_status`. `grape_get_context` reuses the local compile service and returns structured scaffold context-pack items plus Markdown. `grape mcp --print-config` now emits a usable stdio connection shape. | Implement remaining read tools, restricted write tools, restore lookup, final ContextPackItem schema, and MCP safety tests for write boundaries. |
| Doctor/status/debug commands | Tested | `grape status` and `grape doctor` inspect config, database, migrations, Git state, Node runtime, worktree state, and local privacy exclusion. | Add sessions/artifacts/claims/proofs/stale/conflicts/omitted commands as those records become product-facing. |
| Error handling | In progress | CLI commands return documented exit codes for usage/storage/stale states, and doctor reports setup failures. | Add recovery guidance for partial bootstrap, lock conflicts, unsafe compile, missing git metadata, and privacy blocks. |
| Cross-platform behavior | In progress | Core paths normalize separators, Git commands use `execFileSync` args, setup uses Node filesystem APIs, and tests use real temp repos. | Add Windows/WSL path tests and avoid Node runtime assumptions beyond the accepted Node 22.5 SQLite ADR. |
